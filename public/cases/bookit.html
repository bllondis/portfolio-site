What you see here is a small excerpt. BookIT is a large-scale enterprise ERP with two fully separate interfaces - a user-facing booking app and a comprehensive admin console. These six screens are from the admin side only: new modules added to the Office setup section of a live system.

The problem

Managing global office portfolios without the right tools

Before these modules, regional admins managed spaces as rows in a table with no visual reference to physical location. There was no way to see where a space sat on a floor plan, no structured filtering across hundreds of spaces, and no way to manage company events from within the platform. Everything required workarounds or external tools.

The business was growing - more locations, more space types, more events - and the admin console wasn't keeping up. My role was to design the missing functionality while integrating seamlessly into a system thousands of users already relied on every day.

Approach

New modules that feel native to a system with its own language

I worked directly with client requirements and business analysts, translating complex business logic into interfaces that fit naturally within the existing admin console. BookIT had an established Design System, component library, and navigation architecture - every new module had to integrate without disrupting the patterns admins already knew.

The floorplan editor was the most complex challenge: it introduced spatial, map-based interaction into a system built entirely around tables and forms. The work wasn't about inventing from scratch - it was about integrating carefully, making something genuinely new feel like it had always been there.

Spaces - Office setup · Screen 01

Space list - navigation structure & data table

The Office setup dropdown reveals the full scope of the module: Buildings, Spaces, Floorplan editor, Catering, Events, Holidays, Rules - all within one coherent structure. The Spaces table surfaces what admins act on most: name, building, floor, sub-type, type, capacity. With 205 entries across multiple buildings, the layout is designed for fast scanning. The Space wizard handles bulk setup for admins configuring many spaces at once.

TaskGive admins a single place to see and act on every space across all their locations
BenefitFull visibility in one view, with bulk operations and a wizard for fast setup
Hierarchical navigationMulti-column data tableSpace wizardBulk actionsPaginated results

Spaces - Office setup · Screen 02

Space list - expanded filter panel

The filter panel opens alongside the table, keeping results always in view. Each group - Building, Floor, Space type / sub-type, Capacity, Status - has its own searchable list. Active selections appear as chips both in the panel and above the table, so the applied state is always clear without reopening. The hierarchical space sub-type list reflects a real client requirement: the taxonomy across global portfolios includes managed meeting rooms, ideation studios, touchdown spots, lactation rooms, and more.

TaskLet admins narrow hundreds of spaces by building, floor, type, capacity and status simultaneously
BenefitResults stay visible while filters are configured - no context lost
Side filter panelSearch within filterHierarchical taxonomyActive filter chipsReset & Apply

Floorplan editor - Office setup · Screen 03

Floorplan editor - main view

Previously, spaces existed only as data. The floorplan editor gives each space a physical address on a real floor plan image. Pin markers use type-specific icons; the sidebar lists all spaces with colour-coded type indicators and search. Building and floor selectors allow instant switching between locations. The Published badge tracks configuration state. Bringing spatial editing into a console built around tables required careful integration - every element had to feel consistent with the Design System while introducing an entirely new interaction model.

TaskLet admins visually place and manage every space on a real floor plan
BenefitSpaces have a physical location, not just a row in a table
Interactive floor planType-coded pin markersColour-coded sidebarBuilding / floor switcherPublication status

Floorplan editor - Office setup · Screen 04

Floorplan editor - display settings

A dense floor plan with many markers, labels, and gridlines can become hard to navigate. The settings panel gives admins individual layer toggles and sliders for opacity and highlight intensity. The floor plan dims behind the open panel - a live preview of the setting, not an abstract value. All controls use existing Design System components: the configurability is new, but the visual language is familiar.

TaskLet admins reduce visual noise on a dense floor plan without leaving the editing context
BenefitFull control over layers and visibility in one panel
Layer visibility togglesOpacity & highlight slidersLive previewDesign System components

Floorplan editor - Office setup · Screen 05

Floorplan editor - space detail

Selecting a space opens its detail panel in the sidebar - without leaving the editor. The selected space highlights in green on the plan, keeping location visible while the admin edits name, type, capacity, area, department, and two business-critical toggles: whether the space is reservable and whether it has a room display. Navigating to a separate page to edit a space you can see on the map would break the spatial logic entirely - in-context editing was a deliberate and non-negotiable decision.

TaskLet admins edit a space's properties while keeping it visible on the floor plan
BenefitThe spatial relationship between a space and its location is never broken
In-context editingSelected space on mapBusiness logic togglesNo page navigation

Events - Office setup · Screen 06

Events - audience management

Events is a new module that brings company event management into the admin console - something previously handled outside the platform. Admins can create events, send targeted invitations, and manage the full audience without leaving BookIT. The KPI bar gives immediate attendance status: 175 invited, 150 expected, 10 confirmed, 1 declined. The attendee table shows name, email, building, business line, type, and status with colour-coded badges. All actions - search, filter, bulk remove, invite - are on one screen.

Results

What became possible

For the first time, regional admins can see every space in its physical location, manage event audiences, and filter hundreds of spaces - all within one platform. Six modules that previously required workarounds or external tools are now part of a single coherent admin experience, used daily across 40+ countries.

Learnings

What this project reinforced

Integration is harder than invention

Designing for a live system with existing users is more constrained than building from scratch - and demands more precision. Every new pattern has to earn its place.

Constraints sharpen decisions

Working within an established Design System removes stylistic noise and focuses the work on what actually matters: how the interface behaves and what it enables.

Spatial context cannot be interrupted

When users work with a map or floor plan, navigating away to edit breaks the relationship between data and place. Keeping context visible is the whole point.
